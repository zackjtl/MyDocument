OpenBinExclusive

以 GENERIC_READ | GENERIC_WRITE 開啟 .bin，share mode 設為 0（不共享）。
遇到 ERROR_SHARING_VIOLATION 或 ERROR_LOCK_VIOLATION 時重試最多 5 秒。 

// 每個 .bin 代表 65536 個 SN（65536 bit = 8192 byte）
const DWORD kSnBinBytes = 8192;
const int kOpenRetryCount = 200;
const DWORD kOpenRetryMs = 25;

HANDLE OpenBinExclusive(const string& filename, bool createIfMissing)
{
  for (int i = 0; i < kOpenRetryCount; ++i) {
    DWORD disp = createIfMissing ? OPEN_ALWAYS : OPEN_EXISTING;
    HANDLE h = CreateFileA(filename.c_str(), GENERIC_READ | GENERIC_WRITE, 0, NULL,
                             disp, FILE_ATTRIBUTE_NORMAL, NULL);
    if (h != INVALID_HANDLE_VALUE) return h;
    DWORD err = GetLastError();
    if (err == ERROR_SHARING_VIOLATION || err == ERROR_LOCK_VIOLATION) {
      Sleep(kOpenRetryMs); continue;
    }
    return INVALID_HANDLE_VALUE;
  }
  return INVALID_HANDLE_VALUE;
}

DoAccessLocked

在已持有 LockFile 的前提下執行。以 bit map 定位 SN，讀取 1 byte 後做位元運算判斷或標記。
標記後立即 FlushFileBuffers，確保落盤才釋放鎖。 

int DoAccessLocked(HANDLE hFile, uint SN, uint offset, bool checkDuplicate, bool markUsed)
{
  DWORD byteIndex   = (DWORD)((SN - offset) / 8);
  DWORD bitPosition = (DWORD)((SN - offset) % 8);
  if (byteIndex >= kSnBinBytes) return 3;

  SetFilePointer(hFile, byteIndex, NULL, FILE_BEGIN);
  BYTE byte = 0;
  DWORD dwRead = 0;
  ReadFile(hFile, &byte, 1, &dwRead, NULL);

  if (checkDuplicate && (byte & (BYTE)(1u << bitPosition)))
    return 1;   // 重複

  if (markUsed) {
    byte |= (BYTE)(1u << bitPosition);
    SetFilePointer(hFile, byteIndex, NULL, FILE_BEGIN);
    DWORD dwWritten = 0;
    WriteFile(hFile, &byte, 1, &dwWritten, NULL);
    FlushFileBuffers(hFile);
  }
  return 0;
}
回傳值對照
0 成功
1 重複 (duplicate)
2 建立/寫入失敗
3 位移錯誤
4 讀取錯誤
5 檔案鎖失敗
Bitmap 位址計算
byteIndex = (SN - offset) / 8
bitPosition = (SN - offset) % 8
# offset = (SN / 65536) * 65536

AccessSerialNumber（核心入口）

完整流程：開檔 → 初始化（若新建）→ LockFile → ENTER log → DoAccessLocked → EXIT log → Unlock → 關檔。
audit log 寫在 LockFile 持有期間，確保 log 順序與實際互斥區間一致。 

int AccessSerialNumber(uint SN, bool checkDuplicate, bool markUsed,
                         const std::wstring* auditLotDirectory)
{
  bool audit = auditLotDirectory != NULL && !auditLotDirectory->empty() &&
               (checkDuplicate || markUsed);

  int g = SN / 65536;
  uint offset = (uint)g * 65536u;
  string filename = _BaseDir + "\\" + IntegerToString(g) + ".bin";

  // 純檢查且檔案不存在 => 視為未使用，直接成功
  if (!markUsed && checkDuplicate) {
    if (!FileExists(ToWideString(filename).c_str())) return 0;
  }

  HANDLE hFile = OpenBinExclusive(filename, markUsed);
  if (hFile == INVALID_HANDLE_VALUE) return 2;
  if (markUsed && !EnsureBinInitialized(hFile)) { CloseHandle(hFile); return 2; }
  if (!LockFile(hFile, 0, 0, kSnBinBytes, 0)) { CloseHandle(hFile); return 5; }

  if (audit) SnRecordAuditLogEnter(*auditLotDirectory, SN, checkDuplicate, markUsed);
  int ret = DoAccessLocked(hFile, SN, offset, checkDuplicate, markUsed);
  if (audit) SnRecordAuditLogExit(*auditLotDirectory, SN, checkDuplicate, markUsed, ret);

  UnlockFile(hFile, 0, 0, kSnBinBytes, 0);
  CloseHandle(hFile);
  return ret;
}

SnRecordAudit.cpp / .h

ENTER / EXIT Log 寫入

透過 sidecar lock 檔（.append.lock）確保多機台共寫同一份 audit log 時不會出現破碎列。
每列包含：timestamp | stationId | phase | bin | SN | check/record flag | tid | res。 

void SnRecordAuditLogEnter(const std::wstring& lotDirectory, unsigned sn,
                           bool checkDuplicate, bool markUsed)
{
  EnsureStationId();
  wchar_t body[256];
  swprintf(body, 256, L"ENTER | bin=%u | SN=%u | check=%d record=%d | tid=%lu",
           (unsigned)(sn / 65536u), sn,
           checkDuplicate ? 1 : 0, markUsed ? 1 : 0,
           (unsigned long)GetCurrentThreadId());
  LogLine(lotDirectory, body);
}

void SnRecordAuditLogExit(const std::wstring& lotDirectory, unsigned sn,
                          bool checkDuplicate, bool markUsed, int resultCode)
{
  EnsureStationId();
  wchar_t body[256];
  swprintf(body, 256,
           L"EXIT | bin=%u | SN=%u | check=%d record=%d | tid=%lu | res=%d",
           (unsigned)(sn / 65536u), sn,
           checkDuplicate ? 1 : 0, markUsed ? 1 : 0,
           (unsigned long)GetCurrentThreadId(), resultCode);
  LogLine(lotDirectory, body);
}

AnalyzeLogFile 堆疊分析

依 bin 分組後，每組使用「檔案原始順序」做堆疊判斷（不依賴 timestamp）。
追蹤每個 SN 的 ENTER/EXIT 配對，若發現「未 EXIT 前又有不同 station 的 ENTER」，即為 suspicious overlap。 

void AnalyzeRowsForSingleBin(const std::vector<AuditRow>& binRows,
                             unsigned& crossOverlap, unsigned& orphanExit,
                             unsigned& unmatchedEnter)
{
  std::map<unsigned, SnStack> stacks;
  for (size_t i = 0; i < binRows.size(); ++i) {
    const AuditRow& r = binRows[i];
    SnStack& st = stacks[r.sn];
    if (r.phase == "ENTER") {
      if (!st.empty() && st.back().first != r.station)
        ++crossOverlap;          // 不同 station 重入 = 互斥可能失效
      st.push_back(std::make_pair(r.station, r.sn));
    } else if (r.phase == "EXIT") {
      if (st.empty()) ++orphanExit;
      else if (st.back().first != r.station) ++orphanExit;
      else st.pop_back();
    }
  }
  unmatchedEnter = 0;
  for (auto& it : stacks) unmatchedEnter += (unsigned)it.second.size();
}

重點：分析時「不排序時間戳」。因為各機台時鐘可能不同步，檔案原始順序才是唯一可靠的順序依據。

Token 協調與收尾

開跑時透過 session_create.lock 協調清場；收工時由「最後一台」機器執行 finalize：
拿 finalize_audit.lock → 掃描 tok_ring → 確認只剩自己 → 分析 log → 備份 → 刪除。

if (Result == 1)
  throw CError(ERROR_QC_REPEAT, L"[SN Check Error] : Card Serial Number Duplicated!");
